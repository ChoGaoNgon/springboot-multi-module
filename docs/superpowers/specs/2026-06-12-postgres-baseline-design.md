# Design Spec — Re-baseline to PostgreSQL with a minimal User skeleton

> Ngày: 2026-06-12. Xây trên nền sub-project A (Spring Boot 3.3.5 + Java 21, nhánh `feature/springboot3-upgrade`).

## 1. Bối cảnh (Context)

`point-management-sys` là base template (`jp.co.htkk`) đã được nâng lên Spring Boot 3.3.5 / Java 21 / MyBatis ở sub-project A. Domain demo (point-management: step, daily/monthly/transaction point, dashboard) **không quan trọng** cho mục đích base. Chủ dự án muốn:

1. **Chuyển DB production từ MySQL sang PostgreSQL** (thay thế hoàn toàn, không dual-support).
2. **Teardown toàn bộ domain point**, chỉ giữ một bảng `users` + một **User CRUD tối thiểu chạy thật** làm ví dụ.
3. **Test vẫn dùng H2**, ở chế độ **MODE=PostgreSQL** để khớp với production mới.

Kết quả mong muốn: một skeleton multi-module sạch, chạy được ngay trên PostgreSQL, có sẵn ví dụ CRUD hoạt động, dùng làm điểm khởi đầu cho các dự án mới.

## 2. Quyết định đã chốt (qua brainstorming)

| Hạng mục | Quyết định |
|---|---|
| DB target | **PostgreSQL** (thay thế MySQL hoàn toàn) |
| Test DB | **H2 MODE=PostgreSQL** (fidelity với prod) |
| Phạm vi domain | **Teardown toàn bộ point domain**, giữ skeleton + **User** |
| User example | **CRUD tối thiểu chạy thật** (create/get/list) + bảng `users` đơn giản |
| Bảng `user` | Tạo mới `users` (không còn reserved-word `user`) |
| Mapper User | **Viết tay** Postgres-friendly (không regenerate lúc này; tool repoint sang Postgres để dùng sau) |
| Migration tool | **Giữ `mybatis-migrations`**, viết lại script DDL theo Postgres |
| Tên project/package | **Giữ nguyên** `jp.co.htkk` / `point-management-sys` (đổi sau nếu muốn) |

## 3. Teardown — gỡ domain point (mọi module)

Xóa toàn bộ code + schema + test liên quan point-management; đảm bảo không còn tham chiếu (compile sạch):

- **entity**: `Step`, `DailyPoint`, `MonthlyPoint`, `TransactionPoint`, `ChangePointHistory`, `TransactionPointHistory` (+ Example/criteria generated). Giữ/định nghĩa lại `User`.
- **persistence**: generator mappers (`StepMapper`, `DailyPointMapper`, `MonthlyPointMapper`, `TransactionPointMapper`, `TransactionPointHistoryMapper`, `ChangePointHistoryMapper`) + custom mappers (`CustomMonthlyPointMapper`, `CustomDailyPointMapper`, `CustomTransactionPointMapper`). Giữ `UserMapper`.
- **dto**: `admin/dashboard/**`, `monthlypoint/**`, `dailypoint/**`, `transactionpoint/**`. Giữ `admin/user/**` (dọn lại theo CRUD).
- **business-interface / business-implementation**: `DashboardService`, `MonthlyPointService`, `DailyPointService`, `TransactionPointService` (+ impl). Giữ `UserService`.
- **web/api**: `DashboardController` + test dashboard (`DashboardControllerTest`, `DashboardMonthlyIntegrationTest`, `DashboardDailyIntegrationTest`, `DashboardValidationIntegrationTest`). Giữ `UserController` (viết lại).
- **framework enums**: `ETransactionType`, `ETransactionStatus` (point-only). **Giữ `EDeleteFlag`** (soft-delete chung).
- **config**: `endpoint.yml` bỏ entry dashboard.

## 4. Giữ lại — skeleton

- Cấu trúc multi-module (framework, dto, entity, persistence, business-*, web/api, batch, mybatis-generator, mybatis-schema-migration, web parent, root).
- **framework**: exception handling (`ExceptionControllerAdvice`/`DefaultRestExceptionControllerAdvice`/handlers/`ErrorResponse`/`ServiceException`/`MessageService`), validation annotations + validators, constants, converters, security stub (`LoginInfo`), utils, `EDeleteFlag`.
- **web/api config**: `WebMvcConfiguration`, `GlobalsConfiguration` (loader `endpoint.yml`), `SpringdocConfig`, `MyBatisConfig`, `AuditInterceptor`, `AuthorizationInterceptor`, exception config, `AbstractBaseController`, `LoggingAspect`.
- **batch**: `BatchApplication` + khung job (`Invoker`/`JobRunner`/`Job0001`). Nếu `Job0001` phụ thuộc point domain thì rút gọn thành demo generic; nếu độc lập thì giữ nguyên (xác định ở bước plan).
- **mybatis tooling**: repoint generator/migration sang Postgres.

## 5. Domain User mới (CRUD tối thiểu chạy thật)

- **Bảng `users`** (Postgres):
  `user_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, username VARCHAR(255) NOT NULL, email VARCHAR(255) NOT NULL, created_by BIGINT, created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP, updated_by BIGINT, updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP, del_flag SMALLINT NOT NULL DEFAULT 0`.
- **`User` entity**: gọn lại đúng cột trên.
- **`UserMapper` (viết tay XML)**: `insertUser` (`useGeneratedKeys="true" keyProperty="userId"`), `selectById` (lọc `del_flag = 0`), `selectAll` (lọc `del_flag = 0`).
- **`UserService` / `UserServiceImpl`**: `createUser(dxo)`, `getUser(id)`, `listUsers()` — logic thật (thay `searchUser` stub trả null).
- **`UserController`**: `POST /admin/users`, `GET /admin/users/{id}`, `GET /admin/users` (cập nhật `endpoint.yml`); tái dùng `AbstractBaseController` + validation (`@RequiredNotBlank` cho username/email) + exception envelope.
- **DTO**: `UserRequest` (create: `username`, `email`), `UserResponse`, `UserRst`, `UserDxo` — dọn theo CRUD.

## 6. Migration PostgreSQL (config/infra)

- **Driver**: `persistence/pom.xml` đổi `com.mysql:mysql-connector-j` → `org.postgresql:postgresql` (version do Boot 3.3 BOM quản). Bỏ property `mysql.connector.java.version`.
- **`application.yml`** (web/api + batch):
  - `driver-class-name: org.postgresql.Driver`
  - `url: jdbc:postgresql://${POSTGRES_HOST:localhost}:${POSTGRES_PORT:5432}/${POSTGRES_DB:helpo_step}` (bỏ mọi param MySQL: `autoReconnect`, `useUnicode`, `characterEncoding`, `allowMultiQueries`, `useSSL`, `zeroDateTimeBehavior`).
  - `username: ${POSTGRES_USER}` / `password: ${POSTGRES_PASSWORD}`.
  - `mybatis.configuration.jdbc-type-for-null: "NULL"` (**quote bắt buộc** — YAML coi `NULL` không quote là null; đây là gotcha MyBatis+Postgres để tránh lỗi set-null).
  - `pagehelper.helper-dialect: postgresql` (khai báo tường minh).
- **`docker-compose.yaml`**: thay service `mysqlDb` (mysql:3306) bằng `postgres:16` (5432, `POSTGRES_USER/PASSWORD/DB`, volume, healthcheck); cập nhật env app sang `POSTGRES_*`.
- **`mybatis-schema-migration`**: `environments/*.properties` → driver `org.postgresql.Driver` + url Postgres; thay script `20221013045051_create_main_tables.sql` bằng DDL Postgres tạo bảng `users`; `pom.xml` thêm driver Postgres.
- **`mybatis-generator`**: `generatorConfig.xml` → driver/url Postgres, `sqlStatement="JDBC"`, table `users`; `pom.xml` thêm driver Postgres. (Tool, không chạy trong scope này.)
- **README**: cập nhật DB (PostgreSQL), domain (User), lệnh chạy.

## 7. Test harness (H2 PostgreSQL mode)

- **`web/api/src/test/resources/application-test.yml`**: `url: jdbc:h2:mem:testdb;MODE=PostgreSQL;DB_CLOSE_DELAY=-1` (bỏ `NON_KEYWORDS` — `users` không reserved); `driver-class-name: org.h2.Driver`; `jdbc-type-for-null: "NULL"`; `spring.sql.init` nạp `schema.sql` + `data.sql`.
- **`schema.sql`**: `DROP TABLE IF EXISTS users; CREATE TABLE users (... GENERATED BY DEFAULT AS IDENTITY ...)` (Postgres-flavored, chạy trên H2-PG).
- **`data.sql`**: seed vài user.
- **Tests**:
  - **`UserCrudIntegrationTest`** (mới): `POST /admin/users` → `GET /admin/users/{id}` → `GET /admin/users`, assert dữ liệu thật (Controller → Service → MyBatis → H2-PG). Set `.servletPath(...)` để qua `AuthorizationInterceptor`.
  - **`UserValidationIntegrationTest`** (mới): `POST /admin/users` thiếu field → 400.
  - Giữ **`ApiContextLoadsTest`**, **`BatchContextLoadsTest`** (đổi sang H2-PG).
  - Xóa toàn bộ test dashboard/point.
- **`batch/src/test/resources/application-test.yml`**: H2 MODE=PostgreSQL.

## 8. Verification (end-to-end)

1. `mvn clean install -pl -mybatis-generator,-mybatis-schema-migration` xanh trên Java 21; toàn bộ test H2-PostgreSQL xanh.
2. `docker-compose up` (Postgres) → chạy `web/api`: Swagger UI **200**, và **User CRUD chạy thật** qua curl: tạo user (POST) → lấy theo id (GET) → list (GET) trả đúng.
3. `batch` context-loads test xanh (H2-PG).

## 9. Ngoài scope

- Đổi `groupId`/`artifactId`/package (`jp.co.htkk` / `point-management-sys`) — giữ nguyên.
- Module security (sub-project B) — vẫn để sau.
- Chạy lại MyBatis Generator trên Postgres (chỉ repoint config; User domain viết tay).

2. Chuyển sang **writing-plans** lập implementation plan chi tiết (cần liệt kê chính xác file teardown + file mới).
3. Sau đó → sub-project **B (module security)**.
